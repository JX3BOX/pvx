# PVX 多屏与移动端适配规范

## 1. 文档定位

本文以已验收的捏脸数据、体型数据页面为实现基线，统一说明 PVX 在大屏桌面、常规桌面、平板、手机 Web、小程序 WebView 和 App WebView 中的适配方式。

通用视觉、容器、卡片和设计令牌继续以 `docs/design/pvx-visual-style-guide.md` 为准；本文专门描述屏幕变化、终端分流、布局降级和多屏验收。

## 2. 先区分两种“移动端”

PVX 同时存在两套机制，不能混用：

### 2.1 响应式 Web

同一个 Vue 页面通过 CSS 断点适配桌面、平板和手机浏览器。捏脸/体型普通 Web 列表、详情和解析页属于这一类。

响应式判断只关心可用宽度、输入能力和方向，不改变接口、路由语义或业务状态。

### 2.2 小程序/App 专用页面

路由通过 `isMiniProgram() || isApp()` 判断运行环境，并为列表和详情加载 `views/<module>/miniprogram` 下的专用页面。这是终端能力分流，不是 CSS 断点。

缩窄桌面浏览器不会自动进入小程序页面；小程序 WebView 即使物理宽度较大，也仍使用专用页面。业务代码不得用 `window.innerWidth` 模拟终端判断。

Face / Body 当前分流如下：

| 场景 | 普通 Web | 小程序/App |
| --- | --- | --- |
| 列表 | `views/face|body/List.vue` | `views/face|body/miniprogram/List.vue` |
| 详情 | `views/face|body/Single.vue` | `views/face|body/miniprogram/Single.vue` |
| 文件解析 | `views/face|body/Parse.vue` | 仍为显式 Web 路由，不自动替换 |
| 移动数据查看 | 不作为常规入口 | `FaceData.vue` / `BodyData.vue` 显式路由 |

## 3. 断点基线

项目断点以 `@jx3box/jx3box-common/css/var.less` 中的变量为准，新样式必须引用变量，不复制魔法数字。

| 屏幕区间 | 当前变量/范围 | 主要策略 |
| --- | --- | --- |
| 手机 | `@phone`，当前 `≤ 720px` | 单列、触控优先、操作重排 |
| 竖屏平板 | `721px–@ipad-y`，当前至 `1023px` | 主内容单列或紧凑双区块 |
| 横屏平板 | `1024px–@ipad`，当前至 `1133px` | 保留工具栏一行，内容弹性收缩 |
| 小屏桌面 | `1134px–1440px` | 完整功能，卡片区允许局部横向浏览 |
| 常规桌面 | `> 1440px` | 完整布局和稳定内容密度 |
| 大屏桌面 | `@smallpc` 附近及以上，当前 `1680px+` | 增加可见内容，不拉大基础间距 |

注意：项目 `src/assets/css/var.less` 中关于外部断点的注释可能滞后，实际编译值以公共包变量为准。升级公共包后应重新核对断点表现。

断点只用于布局发生结构变化的地方。内容本身能通过 Grid、Flex、`minmax()`、`min-width: 0` 自适应时，不应新增额外断点。

## 4. 页面容器

沿用全站视觉规范中的容器基线：

| 场景 | 最大宽度/内边距 |
| --- | --- |
| 内容最大宽度 | `1440px` |
| 常规桌面水平内边距 | `32px` |
| 中等屏幕水平内边距 | `24px` |
| 手机水平内边距 | `16px` |
| 常规桌面垂直内边距 | `32px` |
| 手机垂直内边距 | `20px` |

`1440px` 是普通内容页默认上限，不是所有页面的强制上限。信息密集型列表或详情经业务评审需要利用大屏空间时，可以在模块专有根节点内将内容容器改为全宽，但必须继续使用上述桌面、平板和手机安全边距；不得修改 `PvxPageShell` 的公共默认值，也不得让未改造页面被动变宽。

主容器必须满足：

- `box-sizing: border-box`，宽度计算包含内边距；
- 子项可收缩时设置 `min-width: 0`；
- 页面级禁止横向滚动；
- 只有卡片带、表格、参数 Tab 等明确内容容器可以局部横向滚动；
- 首屏保持合理最小高度，但移动端避免仅依赖传统 `100vh` 造成浏览器地址栏跳动；新代码优先评估 `100dvh` 并保留兼容回退。

## 5. 普通 Web 列表页

### 5.1 桌面端

- 筛选、搜索、解析和发布入口位于同一工具层级。
- 卡片宽度保持稳定，列数根据内容区实际宽度计算。
- 大屏解除容器上限后，应同步提高承载复杂信息卡片的最小宽度，避免只增加列数导致卡片内容过密；优先使用 `auto-fill + minmax()`，而不是按视口硬编码列数。
- 总览卡片带在空间不足时只允许自身横向滚动，不能撑宽页面。
- 全部列表使用 Grid/Flex 自动排布，最后一项不能裁切或被 `space-between` 拉开异常间距。
- Face 与 Body 通过 CSS 自定义属性表达卡片高度和图片比例差异，不复制整套布局。

### 5.2 平板端

- 页面内边距收紧到约 `20px`，分区内边距收紧到 `16px`。
- 业务 Tab 不再强制 Sticky，避免占用有限高度。
- 竖屏时工具栏允许换行，搜索输入框保持可用最小宽度。
- 横屏平板保持“类型 + 筛选 + 弹性搜索框 + 操作入口”同一行，搜索框承担剩余宽度。
- 卡片总览从多列区块降为单列区块，卡片本身不随容器无限拉伸。

### 5.3 手机 Web

- 页面与分区采用单列，工具栏按“类型、筛选/搜索、主要操作”重排。
- 页面级 Hero 的服务器、区服等公共选择器移动到标题下方并占满可用宽度；条件只维护一份状态，不能在各子页重复初始化。
- 类型选择可以局部横向滚动，选项不可被压缩到不可点击。
- 主要操作按钮占满可用宽度。
- 卡片列表居中展示稳定卡片宽度；如果业务需要双列，必须先验证 320px 宽度和长语言。
- 信息看板在手机端优先改为单列完整卡片；卡片本身已经能够完整展示时，不保留无意义的横向滚动、滚动条或切换箭头。
- 区块标题右侧的低频管理操作可以仅显示图标，点击区不得小于业务验证后的紧凑尺寸，并必须提供可访问名称。
- 筛选浮层宽度不得超过 `100vw - 32px`，相对视口居中。
- 分页器可隐藏不适合手机的 total、jumper 等信息，保留明确的上一页、下一页或加载更多能力。
- 英文、越南文标签允许两行，卡片为第二行预留高度，不能靠缩小字号硬塞。

## 6. 普通 Web 详情页

### 6.1 桌面端

- 预览区、购买/下载区、作者信息和相关推荐保持明确主次关系。
- 图片容器有固定比例或稳定高度，切换图片不引发布局跳动。
- 购买和下载是主要操作，不能被装饰图、长标题或标签遮挡。
- 相关推荐按稳定卡片宽度排列。

### 6.2 平板端

- 详情主内容允许换行，购买区降到预览区下方并占满内容宽度。
- 装饰性图片可以隐藏，业务价格、权限和下载信息必须保留。
- 参数分类和相关推荐只在各自容器内横向滚动。
- 购买按钮和复制码区域改为垂直排列，触控区域保持稳定。

### 6.3 手机 Web

- 作者头部改为单列，可隐藏不影响识别的装饰头像，但保留作者名称和必要身份信息。
- 顶部导航允许换行，低频管理员操作可收进菜单或在手机布局中隐藏；业务主操作不能直接消失。
- 价格和游戏内收费提示允许换行并居中展示。
- 轮播宽度使用可用内容宽度，左右保留安全边距。
- 相关推荐采用横向卡片带时必须显示可滚动线索，并保证页面本身不横向滚动。
- 弹窗、确认框、图片预览和下载提示均需单独验证触摸操作。

## 7. 数据解析页

Face / Body 共用 `ParsePage.vue` 外壳，差异通过 `type` 和对应语言键传入。

- 桌面端保持“导入说明—上传区—能力提示—解析结果”的清晰顺序。
- 上传区域需要足够点击面积，并同时支持点击与拖拽时的状态反馈。
- 平板端参数网格减少列数，标题或总价占独立行，不能挤压数值。
- 手机端上传区收紧但高度不低于可操作阈值；提示卡改为单列。
- 手机端参数 Tab 可在自身容器横向滚动，结果项保持标签、数值和控件对齐。
- 隐私提示可收为图标，但必须提供 tooltip、aria-label 或等价可访问说明。
- 文件选择、解析失败、清空重选和第三方 facedat 组件均要验证。

## 8. 小程序/App 列表页

专用列表采用全屏滚动容器、悬浮操作栏和底部导航预留空间：

- 列表主滚动区与抽屉滚动区职责分离，打开抽屉时避免背景穿透滚动。
- 底部至少预留 `@pvx-mini-bottom-nav`，当前为 `3.5rem`，并叠加宿主安全区。
- 角色类型与筛选通过抽屉完成，确认和重置按钮保持稳定位置。
- 总览与单分类复用同一卡片语义，但可以使用适合触摸的不同视觉组件。
- 无限加载必须区分 loading、完成、空数据和失败状态，不能重复请求。
- 小程序搜索跳转属于与原生端的协议，参数名称和取值不可因界面翻译随意改变。

## 9. 小程序/App 详情与数据页

- 详情页以全宽图片、标签、介绍、数据入口、作者和其他作品组成单列信息流。
- 首图使用 `object-fit: cover` 时必须接受关键内容裁切评审；作品预览不适合裁切时改用 `contain`。
- 页面底部为宿主导航和安全区留白，主要操作不可贴住 Home Indicator。
- 复制码、查看数据等操作通过抽屉承载时，抽屉必须支持关闭、失败反馈和无数据状态。
- 数据页的分类 Tab 可横向浏览；滑杆必须有标签、当前值和足够触控高度。
- 使用 `prefers-color-scheme: dark` 适配宿主暗色模式，不能只反转文字而遗漏卡片、边框、图标和遮罩。
- 小程序/App 专用组件仍需遵循 `docs/common/i18n-development-guide.md`，不能保留仅中文可用的交互文案。

## 10. 图片、触控与可访问性

- 触控目标建议不小于 `44 × 44px`；相邻目标保留足够间距。
- Hover 样式仅放在 `@media (hover: hover) and (pointer: fine)` 中，触屏不能依赖 Hover 才展示操作。
- 图片使用明确的 `aspect-ratio`、宽高或容器约束，避免加载前后跳动。
- 非装饰图片提供有效替代文本；纯装饰图片使用空 alt。
- 键盘焦点不能被 Sticky 导航遮挡，弹层必须有正确的焦点进入与返回。
- 状态不能只用颜色表达，需同时提供文本、图标或形状。
- 支持系统减少动态效果偏好时，应停用非必要位移和缩放动画。

## 11. 长语言适配

每次响应式验收至少覆盖中文、英文和越南文：

- 标题可换行或省略，但不能挤掉主要操作；
- 按钮优先弹性宽度，不设置仅适合中文的固定宽度；
- 标签组允许换行并保持垂直间距；
- 表单标签与输入区在窄屏可改为上下排列；
- 使用 `html:lang(en)`、`html:lang(vi)` 的差异规则时，必须限定在模块根节点下；
- 文案溢出应先调整布局，再考虑截断，不能把业务关键信息截断到不可理解。

## 12. 样式组织

Face / Body 当前样式职责如下：

| 路径 | 职责 |
| --- | --- |
| `assets/css/common/face-body/pc` | 普通 Web 共用列表、详情和变量 |
| `assets/css/common/face-body/parse.less` | 普通 Web 共用解析页 |
| `assets/css/common/face-body/miniprogram` | 小程序/App 共用列表、详情、数据和暗色模式 |
| `assets/css/face`、`assets/css/body` | 模块入口及真实差异 |
| `assets/css/modules/face-body-theme.less` | 已验收现代主题和长语言响应式修正 |

新增规则必须挂在模块根节点下。普通 Web 与小程序样式入口保持独立，不互相引用终端专用变量。只在三个以上页面稳定复用的值才提升为公共令牌。

## 13. 建议验收矩阵

| 类型 | 建议视口/环境 | 重点 |
| --- | --- | --- |
| 大屏桌面 | `1920 × 1080` | 最大宽度、内容密度、间距 |
| 常规桌面 | `1440 × 900` | 完整工具栏、卡片列数 |
| 小屏桌面 | `1280 × 800` | 局部滚动、无页面溢出 |
| 横屏平板 | `1133 × 744`、`1024 × 700` | 工具栏同排、详情降级 |
| 竖屏平板 | `834 × 1194`、`768 × 1024` | 单列、弹层、滚动区域 |
| 手机 Web | `390 × 844`、`414 × 896` | 单列、长文本、触控和安全边距 |
| 最窄手机 | `320 × 568` | 最小可用宽度、无横向溢出 |
| 小程序/App | 真实宿主 + 模拟器 | 终端分流、底部导航、抽屉、暗色模式 |

除固定视口外，还需拖动检查 `720/721px`、`1023/1024px`、`1133/1134px` 和 `1440/1441px` 两侧，避免断点跳变、内容闪烁或临界宽度溢出。

## 14. 页面验收清单

### 布局

- 页面没有横向滚动，允许滚动的局部容器边界明确。
- 卡片不裁切、不异常拉伸，最后一项可完整访问。
- Sticky、Fixed、弹层和宿主导航互不遮挡。
- 横竖屏切换后布局和滚动位置仍可用。

### 内容

- 中文、繁体中文、英文、越南文无关键内容溢出。
- 空数据、加载、失败、无权限和下架状态均有明确布局。
- 用户内容超长、图片缺失、接口字段缺失时不破版。

### 交互

- 搜索、筛选、分页/加载更多、Tab、返回和主要操作可触摸、可键盘使用。
- 购买、支付、复制、下载、文件解析和图片预览流程未因重排改变。
- 弹层可打开、关闭、取消和提交，背景不会穿透滚动。

### 终端

- 普通手机浏览器仍使用响应式 Web 页面。
- 小程序/App 正确进入专用列表和详情。
- 小程序/App 底部安全区、暗色模式和原生跳转协议正常。
- 第三方组件在各断点内可滚动、可操作且没有越界。
